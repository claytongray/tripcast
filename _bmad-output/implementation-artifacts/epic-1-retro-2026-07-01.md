## What We Shipped

MailerSend wired up and first real email sent; a session-scoped resend-reuse feature (login and signup); spec reconciled against AD-6 via correct-course; a 3-layer adversarial review that caught a real gap; 307 → 310 tests green.

---

## ✅ What Went Well

- **TDD held the line.** Every behavioral change started as a failing test (reuse, expiry-not-extended, signup-reuse). The RED/GREEN cycle caught the exact semantics, and the suite stayed clean throughout.
- **Caught the spec drift by asking the right question.** "Is that written as a referable spec?" turned a silent divergence from an adopted architectural decision (AD-6) into a proper correct-course reconciliation. That's governance working as intended.
- **The adversarial review earned its cost.** The Edge Case Hunter found the signup-path gap that the Acceptance Auditor explicitly (and wrongly) waved through as "fine." Diverse review lenses > one. Single-lens review would have shipped it.
- **Verification refused to trust "it works."** The live check surfaced a real trap — and the kernel reproduction test cleanly separated environment artifact from code bug instead of guessing.

---

## ❌ What Bit Us

- **The IPv6 localhost detour.** A stale tripcast-frontend dev server (from a different checkout) was squatting `[::1]:8001`, and macOS resolves `localhost` to IPv6 first — so the browser hit old code the whole time. Real effort burned theorizing about phpdotenv/opcache before `lsof` revealed two listeners. That was avoidable with the right habit up front.
- **The signup path was missed at build time.** The feature's real seam is the shared CheckEmail interstitial, but reuse was wired only through the login controller. The spec intent ("same-browser resend") always covered signup — the implementation didn't. Review caught it; design should have.
- **MailerSend friction landed late.** The API key was copied from another project (`cruisin4parts`), producing confusing 422s. And the daily digest is blocked by the List-Unsubscribe paid-gate — a genuine go-live blocker discovered mid-flight and parked.

---

## 💡 Lessons

1. **Bind/hit `127.0.0.1` explicitly and `lsof -i:PORT` first when local behavior looks wrong** — rule out stale/duplicate servers before debugging code. (Two servers running simultaneously is intentional; just need the right habit.)
2. **For cross-cutting behavior, sweep all entry points at design time** — "who else calls this / who else renders this UI?" The signup gap was a one-line design oversight, not a coding error.
3. **Make "does this touch an adopted AD?" a standard build-time checkpoint** — spec-first governance paid off here.
4. **Label third-party keys in `.env` with their account/owner** — stale key confusion is easy to avoid.
