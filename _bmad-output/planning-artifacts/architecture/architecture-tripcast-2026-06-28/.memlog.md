- (constraint) [ADOPTED PRD] Provider abstraction: weather behind a Laravel service class, fetched by coordinates only; geocoding decoupled from weather (swap weather = config change, zero geocoding change). WeatherAPI's own geocoding unused.
- (constraint) [ADOPTED PRD] Auth = Magic Link only, no passwords, no reset. Long-lived Session via cookie refreshed on activity. Email action links (unsubscribe/end-trip/feedback) work with NO login, signed/scoped to one Trip.
- (constraint) [ADOPTED PRD] Daily send: single Laravel Scheduler command at 9:00 AM America/New_York (DST-tracking). Idempotent per Trip per date via Email Log (no duplicate sends). Per-Trip failure retries <=3x within run, else logs and defers to next day; never sends a broken/empty forecast.
- (constraint) [ADOPTED PRD] Cadence: Welcome Email immediately on trip creation; then quiet until Forecast Window opens (7 days before Departure); Daily Digest every morning from window-open through Return Date; stop after Return Date (Trip -> completed). Paused trips send nothing.

- (direction) Scaling posture [ADOPTED PRD NFR]: design-for-future, build-for-small. v1 = single daily command for low-tens of users. The single command is the named pivot point to chunked/queued processing under load. Architecture must not preclude that seam.
- (decision) [ADOPTED] Paradigm: Laravel layered + provider ports + pipes-and-filters send pipeline. Domain logic in Action classes over Eloquent models; the 3 external providers (weather, geocoding, mail) sit behind explicit PHP interfaces (ports) bound in a ServiceProvider — the only hexagonal touch, placed exactly at the swappable I/O boundary. Daily send is an explicit staged pipeline. Dirs: app/Actions, app/Models, app/Services/{Weather,Geocoding}, app/Digest, app/Console/Commands, Http/Controllers thin; resources/js/Pages (Inertia SSR); resources/views/emails (Blade). Binds=all. Prevents=ad-hoc provider coupling and logic scattered across controllers.
- (decision) Idempotency authority = claim-first DB unique constraint on email_logs(trip_id, send_date). Insert row (status=sending) BEFORE fetch/send; unique index physically rejects a duplicate. Retry updates same row; final failure => status=failed (next day is a new send_date => new row). Binds: daily send, any retry/backfill. Prevents: double-send across retries, crash-reruns, and future concurrent queued workers.
- (decision) Per-trip send is Job-shaped from day one (Clayton expects queued/async soon). Daily Console Command = select-due-trips + dispatch only; the per-trip work (claim, fetch forecast, render, send, log outcome, retry<=3x) is a self-contained dispatchable Laravel Job. v1 may run the queue sync or via worker — that's config, not structure. This IS the named scaling seam, built open. Binds: app/Digest pipeline, app/Console/Commands. Prevents: a monolithic inline loop that can't be parallelized later.
- (decision) Auth links: dedicated single-use login_tokens table (token hash, expiry, consumed_at) for magic-link login; Laravel signed URLs (stateless, no storage, scoped to trip id in route) for idempotent email actions (end-trip/unsubscribe/feedback). Sessions are long-lived cookie-based, refreshed on activity. Binds: auth, email-action handlers. Prevents: replayable login links; and a token table for actions that don't need one.
- (decision) Trip state machine: single owner — ALL status transitions (active<->paused by user, ->completed by system, ->completed/ended by email end-trip link) funnel through one Trip transition method/Action; no surface writes Trip.status directly. Completion is a DEDICATED daily sweep that transitions ANY trip past its Return Date to completed regardless of active/paused (status-agnostic), separate from the send pipeline. Binds: dashboard, email-action handler, daily job, admin. Prevents: three features mutating status incompatibly; paused-past-return trips lingering forever.
- (decision) Queue driver = Redis (Clayton wants real async). Per-trip send Jobs dispatched to Redis queue, processed by a Forge queue-worker daemon. Redis is now a v1 infra dependency on the Forge box. Binds: Digest jobs, deployment. Prevents: sync-inline sends; gives real queue exercise before scaling.
- (decision) Time model (two pinned reference frames). The 7-day forecast rows render in the DESTINATION's local calendar days exactly as WeatherAPI returns them. Binds: cadence calc, completion sweep, digest renderer. Prevents: cadence-calculator and forecast-renderer choosing different reference timezones -> off-by-one day / wrong-day forecasts.
- (decision) No orphan trips. Pre-account trip setup carried through email-capture in the server session; User-upsert + Trip-insert in ONE DB transaction once email given. trip.user_id NOT nullable. Binds: landing/signup flow. Prevents: half-created trip rows needing garbage collection.
